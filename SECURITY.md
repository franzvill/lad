# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this specification or reference implementations, please report it responsibly:

1. **Do not** open a public GitHub issue
2. Email the maintainers directly (see repository contacts)
3. Include detailed steps to reproduce
4. Allow reasonable time for a fix before public disclosure

## Security Considerations

LAD-A2A operates in hostile network environments. The specification includes mandatory security requirements:

- TLS for all communications
- Signed AgentCards with verifiable identity
- User consent before agent connections
- Explicit capability scoping

See the specification for full security requirements.
